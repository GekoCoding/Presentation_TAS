C'est quoi un speedrun

Un speedrun est un pratique dans laquelle le but et de finir rapidement une tâche dans un jeu ou simplement le jeu.​
Le déroulement d’une run (partie) est souvent planifier en avance. Il faut au préalable rechercher la meilleur route à suivre dans un jeu appelée routing (The Legend of Zelda (OFT, BOTW, TOTK).
Une fois la route trouver les speedrunners vont alors tenter d’exécuter leur chemin, le plus efficacement possible pour atteindre le meilleur temps.​

Pour battre un record, les joueurs alternent entre amélioration de la route ou à la recherche de nouvelle route.​ Lors de cette recherche on peut trouver des raccourcis, ou même d’avancer dans le jeu sans avoir accompli tous les objectifs requis prévus par les devs.​ Ce détournement du déroulement du jeu requiert souvent l’exploitation d’un bug (problème dans le code du jeu) ou d’un glitch (problème avec la console).​

L’exploitation de ces erreurs de programmation peut aussi permettre aux speedrunners de faire atteindre aux personnage des vitesses ou valeurs de dégâts impossible à atteindre normalement. Pour déclencher certains de ces bugs il faut parfois manipuler l’état interne de la mémoire de la console de jeu. Ce qui exige des actions très précises de la part du joueur. Par exemple la duplication d’item dans BOTW ou TOTK.​

Les catégories du speedrun

Any % : le but est de terminer le jeu le plus rapidement possible avec ou sans bug ou glitch, on peut ignorer des parties du jeu pour terminer celui-ci.​

100 %: C’est le but de terminer à 100 % le jeu (tout récupérer dans un jeu, faire tout les boss).​

Glitchless : C’est le but de terminer le jeu sans glitch ou bug, c’est une contrainte que les joueurs ce donne.​

Il y a de nombreuse catégories et défis spécifiques à un jeu, par exemple utiliser des périphérique d’entrée différents (tapis de Dance ou la guitare de guitar Hero)​​
